Overview and approach

To detect and respond to user input, we will use Java's built-in scanner class provided by java.util.Scanner.

The necessary import statement has already been added to the top of our file.

Declaring and initializing a scanner

Declare a local variable of type Scanner and initialize it to read from standard input by passing the static variable System.in to the constructor.

Reading and responding to player input

You've already printed the initial board for the user. Now prompt them to enter a row and column. for their move. Read their input and use those values to mark the intended row and column for player 1. Finally, display the updated board.

Since we are using integers for the row and column, you are encouraged to use the nextInt method of the scanner. This will read each integer of input, one at a time, allowing you to capture each to a variable.

Scripting a conversation

The conversation with the player should match the format shown below. NOTE: the prompt mark, '>', indicates where the player is to provide input, followed by sample data from the player.

Here, the line '> 1 2' indicates the player is responding to the prompt with values 1 and 2. You should not make any attempt to print the prompt mark or the user data when printing your output.

[0, 0, 0]

The prompt for player 1 should be entered exactly as shown. This is to facilitate automatic testing. In the final game, you are welcome to modify the prompts as you wish.
